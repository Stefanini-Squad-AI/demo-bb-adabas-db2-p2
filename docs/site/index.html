🎯 Objetivo da Modernização

Eliminar o acesso direto ao Adabas (arquivo ADABAS-SOCIOS, FNR 24) pelos programas Natural, delegando persistência à camada COBOL/DB2.

Arquitetura legado:

Programa Natural → VIEW Adabas → FIND / STORE

Arquitetura alvo (implementada em STFPCS00):

Programa Natural → STFSC00L (LDA) → CALLNAT COBOL → DB2 (SOCIO / SOCIO_PAGAMENTO)

A camada COBOL centraliza:

  • Execução SQL e validação de SQLCODE
  • Conversão de tipos lógicos (Y/N)
  • Cursor para leitura de pagamentos periódicos
  • Return codes padronizados (+000, +100, +803, +999)
  • Encapsulamento das regras de persistência DB2

🏗️ Princípios Arquiteturais

  • Natural permanece como camada de apresentação e orquestração (mapa, PF-KEY, validações de tela).
  • COBOL concentra acesso DB2, cursores e mapeamento SQLCODE → return code.
  • LDA STFSC00L espelha copybook STFSC00B para equivalência funcional.
  • Grupos periódicos Adabas convertidos em tabelas filhas relacionais (proibido flatten).
  • Gerar serviços COBOL apenas para operações encontradas no fonte (FIND→C, STORE→I).
  • Datas na interface em YYYY-MM-DD; não mover campos tipo D do Natural diretamente.

🚨 Riscos Técnicos

  • VERVALOR externo: tarifas de mensalidade não versionadas neste repositório.
  • Transação parcial: inclusão de sócio + 12 pagamentos sem ROLLBACK explícito em falha intermediária.
  • Escopo limitado: apenas inclusão/consulta; alteração e baixa de contrato ainda não modeladas.
  • Paridade Adabas/DB2: diferenças de semântica em campos nulos (DATA_BAIXA, HORA_BAIXA).
